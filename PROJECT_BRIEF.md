# 语音输入法 - 项目说明书

## 项目目标

打造一款 Windows 11 桌面常驻语音输入助手，让用户通过"按住说话、松开输入"的方式快速将语音转为文字并注入当前输入框，提升输入效率。

## 为什么做

### 痛点分析

1. **传统打字效率低**：长文本输入耗时费力，尤其在会议记录、文档撰写场景
2. **特殊场景需求**：驾驶、做饭等场景无法方便地打字
3. **现有方案不足**：
   - Windows 自带语音输入体验一般
   - 第三方输入法隐私顾虑大
   - 专业语音软件价格昂贵

### 价值主张

- **效率优先**：按住即说，松开即输，零延迟体验
- **隐私安全**：本地常驻，数据可控
- **体验流畅**：精致 HUD 动画，实时反馈
- **功能实用**：支持多语言、可选 LLM 纠错

## 面向谁

### 主要用户群体

1. **内容创作者** - 作家、记者、博主，需要快速记录灵感
2. **商务人士** - 会议记录、邮件回复、报告撰写
3. **程序员** - 代码注释、文档编写，支持技术术语纠错
4. **特殊场景用户** - 驾驶者、家务劳动者，解放双手

## 第一版做什么

### 核心功能（MVP）

1. **系统托盘常驻**
   - 启动后仅显示托盘图标，无主窗口
   - 右键菜单提供语言切换、LLM 设置、退出等

2. **全局热键激活**
   - 默认激活键：**Right Shift（右 Shift）**
   - 按住开始录音，松开结束识别
   - 全局有效，不依赖窗口焦点

3. **语音识别**
   - 使用火山引擎/豆包语音云端流式 ASR
   - 默认语言：简体中文（zh-CN）
   - 支持语言切换：简体中文、English、繁體中文、日本語、한국어
   - 流式识别，实时显示中间结果

4. **屏幕底部胶囊 HUD**
   - 无边框、置顶、不抢焦点
   - 左侧 5 根竖条波形动画（真实麦克风 RMS 驱动）
   - 右侧实时文本显示（单行 + 最大宽度 + 超出省略号）
   - 入场弹簧动画、退场淡出动画

5. **文本注入**
   - 备份剪贴板 → 写入识别文本 → 模拟 Ctrl+V → 恢复剪贴板
   - 容错处理：注入失败时保留结果供手动复制

6. **LLM Refinement（可选）**
   - OpenAI 兼容 API，支持自定义 Base URL、API Key、Model
   - 保守纠错：中文谐音、中英文混输、技术术语修正
   - 不改写句子、不润色、不删除正确内容
   - 未配置时不影响主流程
   - 更适合作为增强模式，而不是默认实时主链路

### 技术栈

- **框架**：C# + .NET 8 + WPF
- **语音识别**：火山引擎/豆包语音云端流式 ASR
- **LLM**：OpenAI 兼容 API
- **Win32 互操作**：全局键盘钩子、剪贴板操作、输入注入

## 暂时不做什么

### 后续版本规划

1. **真正 IME 集成** - 不注册为系统输入法，不做 TSF 文本服务
2. **本地语音识别** - 不做本地 Whisper、Windows Speech API
3. **高级 IME 兼容** - 深度 IME 兼容放后面，第一版用剪贴板方案
4. **自定义热键** - 第一版固定 Right Shift，后续支持修改
5. **开机自启** - 托盘菜单预留，第一版不实现
6. **云端同步** - 多设备同步、云端备份
7. **自定义图标** - 使用默认图标

## 风格目标

### 设计原则

1. **简洁至上** - 托盘常驻，无主窗口，核心功能一触即达
2. **反馈及时** - 波形实时可视化，文本实时显示
3. **容错友好** - 各环节失败不崩溃，局部降级
4. **性能优先** - 启动快、内存低、电量省

### 视觉风格

- **HUD**：圆角胶囊、轻量动效、现代克制
- **动画**：从底部轻弹出现、平滑过渡、自然退场
- **配色**：系统级工具风格，接近 Windows 11 原生体验

## 当前实现备注

- 当前版本已经可以完成主链路闭环
- 产品重点是实时输入体验，不追求复杂功能堆叠
- 后续更值得投入的是稳定性、连续输入体验和热键兼容性
